## ⚠️ Why Most Beginners Fail

Most beginners fail at programming because they **directly jump into code** without understanding the basics and fundamentals.

```
Wrong approach ❌
Jump straight into code → get confused → give up

Right approach ✅
Understand computers first → understand languages → then write code
```

> Strong fundamentals are crucial to learning and grasping any programming language quickly.

---

## 🖥️ What is a Computer?

A computer is any device that processes input and gives output.

```
Mobile phone  =  computer
Laptop        =  computer
Desktop       =  computer
```

**Components of a Computer:**

| Component | What it does |
|-----------|-------------|
| CPU / Processor | Brain of the computer — executes all tasks |
| RAM | Temporary memory — stores data while running |
| Hard Disk | Permanent memory — stores files permanently |
| Monitor | Output device — shows results |
| Keyboard / Mouse | Input devices — takes your input |

> The most important component is the **Processor (CPU)** — also called microprocessor.
> Every task you do on a computer (playing video, writing a blog, listening to music) is directly linked to the processor.

---

## ⚙️ What is a Processor?

**Definition:**
> The processor is the Central Processing Unit (CPU) of a computer — responsible for executing all tasks.

**Key points:**
- Made using **semiconductor technology** — known for its fast execution capabilities
- Semiconductor technology uses **transistors** and **capacitors** in its manufacturing
- The processor is a **"dumb" device** — it cannot do anything without instructions
- Just like we communicate in English, we need to communicate with the machine in its own language

**Types of Transistors:**

| Type | Full Form |
|------|-----------|
| NPN | Negative-Positive-Negative |
| PNP | Positive-Negative-Positive |

---

## 🔢 Binary Language

Transistors operate with two voltage levels:

```
Low Voltage  (0V)  →  Binary  →  0
High Voltage (5V)  →  Binary  →  1
```

This is why computers only understand **0s and 1s** — called **Binary Language** or **Machine Level Language (MLL)**.

**Definition:**
> Binary language is the most basic language of a computer — made up of only two digits: 0 and 1.

---

## 📋 What is a Program?

**Definition:**
> A program is a group of instructions given to a processor to perform a specific task.

```
Single instruction  →  "Add two numbers"
Many instructions   →  Program
```

> Our main objective as programmers is to write instructions in a way the machine understands
> and that produce the desired output.

**Problem with MLL:**
- Writing in binary (0s and 1s) is very hard for humans
- For small operations it might be manageable
- For large programs it becomes completely impractical

## 💻 Types of Programming Languages

Programming languages evolved over time to make it easier for humans to communicate with machines.

---

### 1️⃣ Machine Level Language (MLL)

**Definition:**
> Machine Level Language is the lowest level language — made of pure binary (0s and 1s) — directly understood by the processor.

```
Example:
01001000 01100101 01101100 01101100 01101111
H        e        l        l        o
```

| Feature | Detail |
|---------|--------|
| Made of | Only 0s and 1s |
| Understood by | Processor directly |
| Human friendly | Very difficult |
| Conversion needed | None |

---

### 2️⃣ Assembly Level Language (ALL)

**Definition:**
> Assembly Level Language uses short human-readable words called mnemonics (like ADD, SUB) instead of binary code.

```
Example:
ADD A, B     <- add two numbers
SUB A, B     <- subtract two numbers
MOV A, 5     <- move value 5 into A
```

| Feature | Detail |
|---------|--------|
| Made of | Mnemonics (short words) |
| Understood by processor | Needs conversion first |
| Human friendly | Somewhat |
| Conversion needed | Assembler converts ALL to MLL |

**What is an Assembler?**
> An assembler is system software that converts Assembly Level Language (ALL) into Machine Level Language (MLL).

```
Assembly Code  →  [Assembler]  →  Machine Code (0s and 1s)
```

> The assembler is system software — built by engineers so we don't have to write in binary!

---

### 3️⃣ High Level Language (HLL)

**Definition:**
> High Level Language uses English words and symbols that are easy for humans to read and write — like print, scan, if, else, +, -, /.

**History:**
- Developed in the **1960s** by researchers
- Used symbols like `+`, `-`, `/`, `x`
- Used English terms like `print`, `scan`, `if`, `else`

| Feature | Detail |
|---------|--------|
| Made of | English words + symbols |
| Understood by processor | Needs conversion first |
| Human friendly | Very easy |
| Conversion needed | Compiler converts HLL to MLL |
| Examples | Java, C, C++, Python, JavaScript |

> HLL is preferred and more human-friendly compared to MLL and ALL.
> However, the processor still only understands binary code — so conversion is always needed.

---

## 🔧 What is a Compiler?

**Definition:**
> A compiler is system software that converts High Level Language (HLL) into Machine Level Language (MLL) so the processor can understand and execute it.

```
High Level Code  →  [Compiler]  →  Machine Code (0s and 1s)
                                         |
                                    Processor runs it
                                         |
                                       Output
```

**Key points:**
- Developed in the **1960s** to bridge the gap between humans and machines
- Before compilers, programmers had to write in binary or assembly — very painful!
- Since the 1960s, compilers are used for **C, C++, Java** and many more languages
- The compiler is also system software — built by engineers, used by us

**Comparison of all conversion tools:**

| Language | Converted By | Output |
|----------|-------------|--------|
| Assembly (ALL) | Assembler | Machine Code |
| High Level (HLL) | Compiler | Machine Code |
| Java specifically | javac compiler | Bytecode then JVM runs it |
